# Contributing to swiftdb

## Getting started

1. Fork and clone the repo
2. Install dependencies: `bun install`
3. Build: `bun run build`
4. Copy to your FiveM server's `resources/` directory and test

## Project structure

```
src/
  index.ts              # Entry point, export registration
  config.ts             # Convar loading, connection string parsing
  types/                # TypeScript interfaces
  core/
    pool.ts             # mysql2 pool, backpressure, connection retry
    queryEngine.ts      # Query execution and prepared statements
    transactionManager.ts
    connection.ts       # Connection acquire/release
    batchExecutor.ts    # Batch query executor
  cache/
    statementCache.ts   # LRU named parameter cache
  compat/
    oxmysql.ts          # oxmysql compatibility aliases
    mysqlAsync.ts       # mysql-async compatibility aliases
    ghmattimysql.ts     # ghmattimysql compatibility aliases
  logger/               # Query logging, slow query warnings
  profiler/             # Sampled MySQL SHOW PROFILES
  utils/
    parseArguments.ts   # Named/positional param conversion
    parseResponse.ts    # Result transformation per query type
    scheduleTick.ts     # FiveM tick scheduling
    typeCast.ts         # mysql2 type casting
```

## Development

- TypeScript with strict mode
- esbuild bundles everything into a single `dist/build.js`
- `fxmanifest.lua` is generated by `build.js` — don't edit it manually
- Target: Node 22, CJS output

## Pull requests

- Keep changes focused — one feature or fix per PR
- Test on a running FiveM server before submitting
- Don't break compatibility with oxmysql, mysql-async, or ghmattimysql exports

## Reporting issues

Open an issue with:
- FiveM server build number
- Database engine and version (MySQL/MariaDB)
- Relevant server console output
- Steps to reproduce
